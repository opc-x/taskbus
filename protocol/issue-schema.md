# Issue 结构标准

> LLM 可消费的最小完备结构。每个字段都必须能被 agent 直接读取执行。

## Master Issue 结构

```markdown
## 靶心
[本圈要让什么从「不存在/不正确」变成「存在/正确」]

## 本圈范围
- 做：[明确列出]
- 不做：[明确列出] — 原因：[为什么]

## Child Issues
- [ ] #N [标题]
- [ ] #N [标题]

## 验收标准
本圈所有 child 关闭 → master 关闭
```

## Child Issue 结构

```markdown
## 目标
[一句话：让什么从「不存在/不正确」变成「存在/正确」]
master: #N

## AC（验收标准）
- [ ] [可验证的功能条件]

## DoD（完成定义）
- [ ] tsc --noEmit → 0 errors
- [ ] lint → 0 warnings

## T1（机器验证）
- `[命令]` → 期望：`[精确值]`

## T2（AI推理验证）
- [判断标准] → 置信度 ≥ [X]%

## T3（人工验证）
- 步骤1：[精确操作]
- 预期：[符合则通过]

## 执行步骤
**Step 1：**
**Step 2：**

## 边界
不做：[out of scope] — 原因：[为什么]
```

## 字段规范

| 字段 | 要求 | 反例 |
|------|------|------|
| 目标 | 一句话，含「从X变成Y」 | ❌「优化性能」|
| AC | 可验证，不模糊 | ❌「用户体验好」|
| T1 | 有命令 + 期望值 | ❌「跑一下看看」|
| 边界 | 必须有「原因」 | ❌「不做登录」|
