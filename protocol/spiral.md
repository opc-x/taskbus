# 螺旋执行协议

## 核心结构

```
        靶心 Vision
            │
    ┌───────┴───────┐
    │   第N圈（当前）  │
    │  决策→需求→执行  │
    │    →验收→反馈   │
    └───────┬───────┘
            │ +1圈
    ┌───────┴───────┐
    │    第N-1圈     │
    └───────────────┘
```

## 五节点定义

| 节点 | 编号 | 职责 | 产出物 |
|------|------|------|--------|
| 决策 | O01 | 定义本圈靶心和边界 | master issue |
| 需求 | O03 | 拆解 child issue，定义 AC/DoD | child issues |
| 执行 | O04-07 | Cursor 并行执行 | PR / 代码变更 |
| 验收 | O08 | T1/T2/T3 验收，关单 | verified issues |
| 反馈 | O10 | 复盘，决定是否升圈 | retro comment |

## 升圈规则

```
本圈所有 child issue 验收通过？
        │
      YES → 复盘 → 命中靶心？
                    YES → 完成
                    NO  → 开下一圈（+1圈）
      NO  → 修复 → 重新验收
```

## 每圈只做加法

- 新圈建立在上一圈的基础上
- 不推翻已验收的 issue
- 靶心可以随业务理解深化而微调，但方向不变
