# 螺旋执行协议

## 完整架构

```mermaid
graph TD
    S["🎯 战略意图"] -->|LLM 语义拆解| TQ{目标队列}
    TQ -.-> T1["目标01"]
    TQ -.-> T2["目标02"]
    TQ ==>|当前激活| TC["★ 当前目标"]
    TQ -.-> T4["目标04"]
    TQ -.-> T5["目标∞"]
    TC -->|进入螺旋| H["🎯 靶心"]

    subgraph spiral["螺旋执行（每圈）"]
        H --> O01["O01 决策"]
        O01 --> O02["O02 需求"]
        O02 --> O03["O03 执行"]
        O03 --> O04["O04 验收"]
        O04 --> O05["O05 反馈"]
    end

    O05 -->|"未命中靶心 → +1圈"| O01
    O05 -->|"命中靶心 → 完成"| DONE["✅ 目标达成"]
```

## 五节点定义

| 节点 | 编号 | 职责 | 产出物 |
|------|------|------|--------|
| 决策 | O01 | 定义本圈靶心和边界 | master issue |
| 需求 | O02 | 拆解 child issue，定义 AC/DoD | child issues |
| 执行 | O03 | Cursor 并行执行 | PR / 代码变更 |
| 验收 | O04 | T1/T2/T3 验收，关单 | verified issues |
| 反馈 | O05 | 复盘，决定是否升圈 | retro comment |

## 升圈规则

```mermaid
flowchart TD
    A{"本圈所有 child issue 验收通过?"} -->|YES| B{"命中靶心?"}
    A -->|NO| C["修复"] --> A
    B -->|YES| D["✅ 完成"]
    B -->|NO| E["开下一圈 +1圈"] --> O01["O01 决策（新圈）"]
```

## 每圈只做加法

- 新圈建立在上一圈基础上，不推翻已验收的 issue
- 靶心可随业务理解深化微调，但方向不变
- 战略意图分解出的目标队列，每次只激活一个进入螺旋
